TPU: 31B Gemma 4 v6e-4, JAX+XLA, int8 quantized Measured

No custom kernels. The entire forward pass is ~500 lines of JAX. XLA compiles it to a single fused while loop on 4 TPU v6e chips. Zero Python in the hot path - the decode loop runs entirely on-chip via jax.lax.while_loop.

Dual-path architecture

Auto-switches based on --max-ctx. <= 32K: single-scan with bf16 KV cache. One jax.lax.scan over all 60 layers with jax.lax.cond dispatch. Fastest for short-to-medium context. > 32K: split-cache with int8 KV cache. 50 sliding-window layers use a 1024-entry circular buffer, 10 global layers use full-context blockwise attention with online softmax. Processed in 10 groups of 6 (5 sliding + 1 global), eliminating jax.lax.cond overhead. Int8 KV with per-head scales provides 50% memory reduction. No compromise -- fast short-context AND 128K support.

EAGLE-3 Speculative Decoding TPU v6e-4, experimental

Single-user latency optimization via speculative decoding. A trained 450M-param draft head proposes K=5 tokens per cycle; the full 31B target verifies all K+1=6 in one forward pass. Lossless for greedy decode. Pipeline validated end-to-end on v6e-4 with training confirmed working.

Training details. Online feature capture: each step runs a full 512-token prefill through the 31B target (3.1s compile, then cached) to extract features at layers 2, 30, 59. TTT (Training-Time Test) loss: 5-depth draft chain where depths 0-1 use target features and depths 2-4 use the draft's own hidden states, training robustness to self-approximation error. Fused train step: loss + grad + clip + AdamW update all on-device via single jitted function with explicit out_shardings for JAX 0.6.2 SPMD compatibility. XLA compilation: 554s first run (2.2GB program), cached thereafter. 5.7s per training step on v6e-4.

Why tau is still low with 1K examples

At loss 7.1, per-depth cross-entropy is ~1.4 nats = ~24% probability on the correct next token. Greedy acceptance requires the draft's argmax to match the target's argmax, which needs >50% probability concentration on the correct token. The EAGLE-3 paper trains on 100K-500K examples for days to achieve tau=3.5-4.5. Our 1K-example proof-of-concept validated the pipeline; scaling to 10K+ examples with self-distilled data (target model's own outputs) is the path to production tau.

GPU: 31B Gemma 4 H100 SXM 80GB, Rust+CUDA, FP8+F16 Measured

FP8 weights + F16 KV cache. Per-channel FP8 weights from the RedHatAI/gemma-4-31B-it-FP8-Dynamic checkpoint with channelscale fused into post-norm kernels. F16 KV cache + F16 paged attention (FA3 SM90 for sliding, SM89 for global). All 60 layers captured in a single CUDA graph (~1400 nodes). 7,943 tok/s peak (B=512), PPL 13.53. Per-layer KV cache sizing enables 128K context.

31B Gemma 4 forward pass (~13 launches per layer)

1.fused_rmsnorm_fp8_quant - input layernorm + FP8 quantize 2.fp8_gemm + f32_to_f16_sat - fused Q||K||V projection 3.vnorm_f16 - parameter-free RMSNorm on V 4.fused_qk_rmsnorm - per-head RMSNorm on Q and K 5.fused_rope_partial_f16kv - partial RoPE + F16 KV cache write 6.paged_decode (FA3) - attention (head_dim=256 sliding, 512 global) 7.quantize_fp8_per_token - attn output to FP8 8.fp8_gemm - O projection 9.fused_norm_add_residual - channelscale + rmsnorm + residual add 10.fused_rmsnorm_fp8_quant - pre-FFN layernorm + FP8 quantize 11.fp8_gemm + f32_to_f16_sat - gate||up projection 12.fused_gelu_mul_fp8_quant - GELU(tanh)(gate) * up to FP8 13.fp8_gemm - down projection 14.fused_norm_add_residual - channelscale + rmsnorm + residual add + layer_scalar

Method

TPU benchmarks (tag v0.3.0):
Hardware: Cloud TPU v6e-4 (4 chips, 128 GB HBM, ~3.3 TB/s aggregate bandwidth), us-east5-b.
Model: google/gemma-4-31B-it, int8 weight quantization (jnp.int8), bf16 activations.
Timing: jax.block_until_ready() after each step. Warmup: 3 steps discarded. Measurement: median of 20 iterations.
LIBTPU flags: --xla_tpu_enable_async_collective_fusion=true --xla_tpu_dot_dot_fusion_duplicated=true --xla_tpu_scheduler_rerun=5
Perplexity: 19.24 (split-cache, int8 KV with per-head scales), improved from 25.51 baseline. Measured on short text passage, compared against HuggingFace transformers bf16 reference on GPU.
Context scaling: 512 to 128K tested. Dual-path architecture auto-switches based on max_ctx: <= 32K uses single-scan with bf16 KV cache; > 32K uses split-cache with int8 KV cache (50 sliding layers in circular buffer, 10 global layers with blockwise attention, BLOCK_K=8192, online softmax).

vLLM GPU comparison (measured):
Hardware: H100 SXM 80 GB on vast.ai ($1.92/hr).
Model: RedHatAI/gemma-4-31B-it-FP8-Dynamic (same 31B Gemma 4, FP8).
Engine: vLLM with default settings, max_ctx=2048.
Batch sizes: 1, 2, 4, 8, 16, 32, 48, 64, 96, 128, 256, 512, 768. Peak: 3,848 tok/s at B=128.

GPU pipeline (v3 codebase, Rust + CUDA):
Hardware: H100 SXM 80 GB.
Model: 31B Gemma 4, FP8 E4M3 weights (per-channel scale), F16 KV cache.
Forward: ~13 kernel launches per layer (post-fusion), 60 layers + lm_head + argmax = ~1400 nodes, captured in a single cuGraphLaunch.
Kernels: cuBLASLt FP8 GEMMs (SM90), FlashAttention-3, custom fused ops (RMSNorm+quant, RoPE+KV, GELU*mul+quant, norm+channelscale+residual+scalar, logit softcap).
Per-layer KV: sliding layers capped at 32 blocks (1024 tokens), global layers get full context. Enables 128K on single 80GB GPU.
