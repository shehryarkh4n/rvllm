Kernels

Every kernel has a pinned variant and a workspace contract. No dispatch fallback chains.

CUTLASS SM90 FP8 GEMM

kernels/cutlass_fp8_gemm.cu, kernels/cutlass_fp8_gemm_residual.cu

40 non-residual variants + 10 residual-fused variants, all templated on (TileShape, ClusterShape, KernelSchedule). Autotuned per shape, result pinned into policy.json (shipped as a build artifact, not a runtime cache).

  • Non-residual: QKV, gate_up, down_proj.
  • Residual-fused: o_proj. One launch writes GEMM(act_fp8, W_fp8) + residual; eliminates one HBM round-trip per layer.
  • Schedule rule: mainloop schedule must match epilogue schedule. KernelTmaWarpSpecialized (WS) mainloop paired with TmaWarpSpecializedCooperative (Coop) epilogue causes CUDA_ERROR_ILLEGAL_ADDRESS only inside graph replay — caught in a CUDA static_assert.
  • v2 dispatch pins o_proj residual to variant v1 (128x128x128 Coop/Coop matched).

FlashAttention-3 SM90 paged decode

kernels/fa3_sm90_wrapper.cu → libfa3_kernels.so (23 MB)

WGMMA + TMA, built from FlashAttention-3 Hopper source. Paged KV layout [2, num_blocks, block_size, num_kv_heads, head_dim], GQA via num_heads / num_kv_heads ratio. head_dim = 128 hard gate.

No PTX fallback. If libfa3_kernels.so is not present, the engine refuses to load.

One decode step, in order

What happens between step_launch and step_collect

step_launch(diff):
  1. schedule(diff)                          CPU, O(N), zero allocs
  2. metadata pack + 1x HtoD                 positions, context_lens, block_tables,
                                             slot_mapping into one packed i32 buffer
  3. graph_pool.replay(bucket = pad_up(N))   one cuGraphLaunch
       embedding_gather                      1 kernel
       for layer in 0..28:                   11 kernels / layer = 308
         fused_add_rmsnorm_fp8_quant
         CUTLASS FP8 GEMM  (QKV)
         fused_rope_kv_write
         FA3 paged_decode
         quantize_fp8_per_token  (post-attn)
         CUTLASS FP8 GEMM + residual  (o_proj, v1)
         fused_rmsnorm_fp8_quant
         CUTLASS FP8 GEMM  (gate_up)
         fused_silu_mul_fp8_quant
         CUTLASS FP8 GEMM  (down_proj)
         residual_add_f16
       fused_residual_rmsnorm  (final)       1 kernel
       cuBLAS HGEMM f32 → lm_head            1 kernel
       argmax                                1 kernel
  4. cuMemcpyDtoHAsync argmax → pinned[w]    4 x N bytes (512 B at N=128)
     cuEventRecord event[w]
  5. return; w ^= 1                          double-buffer flip

step_collect():
  1. cuEventSynchronize event[r]             wait for step-1 DtoH
  2. read pinned[r][0..N]                    CPU reads new tokens
  3. scheduler.commit()                       mark tokens, free finished

Total GPU kernels per decode step: 1 + 28x11 + 1 + 1 + 1 = 311 launches, all captured into 1 cuGraphLaunch. Total DtoH per step: one 4xN byte copy of argmax token IDs. Total HtoD per step: one packed metadata buffer (<100 KB at N=128, max_blocks=129).

Correctness discipline

Explicit rules. Violations fail the build or startup — never degrade silently.

  1. No fallbacks. Missing autotune policy entry for a shape panics with the shape. Missing FA3 .so refuses startup. Missing CUTLASS .so refuses startup. Stale autotune caches from prior deploys are not consulted — policy lives in the SHA-pinned build artifact.
  2. Graph-capture invariant. Metadata buffer layout is frozen per (bucket, max_blocks_per_seq). Captured graphs bind those exact offsets. There is no "non-padded" upload path. Prefill and decode have separate APIs; they do not share metadata offsets. This is the fix for the April 16 CUDA_ERROR_ILLEGAL_ADDRESS regression.
  3. Real block-change detection. Scheduler emits ContinuedRequest::block_table_update: Option<Vec<BlockId>> whenever a sequence's physical block list grows. Worker unions this with CoW copies. Missing either signal leaves stale block_ids in the captured graph — wrong KV reads, silent correctness bug, no crash.
  4. CUTLASS schedule/epilogue pairing. Mainloop and epilogue must match. Mismatched variants cause CUDA_ERROR_ILLEGAL_ADDRESS only inside graph replay. Enforced in v3 as a CUDA static_assert. In v2, dispatch pins the fused residual path to variant v1 (Coop/Coop, hand-verified).
  5. No unwrap() in libraries. Result<T, RvllmError> end-to-end. Errors carry structured context (stream, kernel name, launch config) — not stringified DriverError that loses fault location.

Reproduce

One command, one H100, everything from source

# One-shot: rent an H100 on vast.ai, build from source, run bench, land JSON locally.
VASTAI_API_KEY=<your_key> ./race.sh

# Manual (on an H100 box)
bash kernels/build.sh               # custom PTX (fused norm / silu / argmax / ...)
bash kernels/build_cutlass_so.sh    # CUTLASS SM90 FP8 GEMM .so
bash kernels/build_fa3.sh           # FlashAttention-3 SM90 .so (~10 min)

./target/release/autotune-cutlass   # populates policy.json for your GPU

cargo build --release --features cuda-graphs --bin rvllm-v2-bench
./target/release/rvllm-v2-bench \
  --model Qwen/Qwen2.5-7B \
  --fp8 \
  --n "1,4,8,16,32,64,128" \
  --output-len 512 \
  --iters 3

v3: clean-slate rewrite (in progress)

16-agent spec swarm + 15-crate scaffold with DAG and LoC invariants

v3 collapses the frankenstein dispatch chain into a single execution path, makes CUTLASS schedule mismatches a compile error, enforces one metadata upload path, and introduces a GraphSafe marker trait that makes realloc-inside-capture unrepresentable at the type level. 16 focused specs (v3/specs) consolidated into v3/IMPL_PLAN.md. DAG and per-crate LoC budgets enforced in CI via tests/dag.rs and tests/loc_budget.rs.
