Abstract

Andy Norris (m0at) -- San Francisco, March 28 2026

rvLLM is a from-scratch Rust implementation of the vLLM inference engine for OpenAI-compatible API serving with CUDA GPU acceleration. 23 Rust crates and 15 pre-compiled CUDA PTX kernels, compiling to a single 16 MB static binary. On a single NVIDIA B200 (180 GB), rvLLM achieves 3,946 tok/s serving Qwen2.5-1.5B with zero errors across 13,553 verified requests at batch sizes up to $N\!=\!4{,}096$. On a single A100 SXM4 (40 GB), it reaches 3,191 tok/s -- 16x faster than Python vLLM 0.18 on identical hardware.

The speedup derives from three reinforcing mechanisms: GPU-resident greedy decoding via an argmax kernel that reduces device-to-host transfer by ~32,000x per token, CUDA graph replay that collapses the entire decode forward pass into a single driver call, and a zero-copy memory architecture with no Python objects, no PyTorch tensor metadata, and no garbage collector.

The GPU Argmax Kernel

The single most impactful optimisation

Standard LLM serving transfers the entire logit matrix from GPU to host every decode step. For Qwen2.5-1.5B at batch 128: 74 MB per step. The argmax kernel finds the max-probability token directly on GPU: 512 bytes per step. A ~150,000x reduction.
$\text{DtoH}_{\text{logits}} = 128 \times 151{,}936 \times 4 = 74.2\;\text{MB}$   vs.   $\text{DtoH}_{\text{argmax}} = 128 \times 4 = 512\;\text{B}$

Where the Speed Comes From

Compound effect of seven reinforcing mechanisms

  1. GPU-resident argmax -- eliminates 74 MB DtoH per step (~3 ms at PCIe Gen4)
  2. CUDA graph replay -- one cuGraphLaunch per decode step
  3. Mixed prefill+decode batching -- no pipeline bubbles
  4. No GIL -- parallel scheduling, tokenisation, sampling via Rayon
  5. No GC pauses -- deterministic deallocation via Rust ownership
  6. Zero-copy activations -- all on GPU throughout N-layer forward pass
  7. Prefix caching -- shared system prompts reuse KV blocks

Coherence

Verified on 5 diverse prompts at every batch size

 -> Paris. The capital of France is Paris...

"Write a function that sorts a list in Python:"
 -> def sort_list(list): return sorted(list)

"Explain quantum computing in simple terms:"
 -> Quantum computing is a new type of computing
    that uses quantum mechanics...
